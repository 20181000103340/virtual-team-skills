# 通用行为规范

所有角色在执行任何任务前，必须遵守以下规范：

## 工作纪律

1. **先读取所有输入文件再开始工作**，不要边读边做
2. **严格按照任务书的要求执行**，不做额外的事（不加功能、不重构无关代码、不加注释到未改动的代码）
3. **遵循项目现有代码风格和模式**，通过阅读相邻文件来确认风格
4. **产出物必须写到任务书指定的路径**，不要写到其他位置
5. **产出物必须包含自检结果**（逐条对照验收标准，标记 [x] 或 [ ]）
6. **遇到阻塞时记录到报告中**，不要停止或猜测

## 安全约束

7. **不要修改任务书中标注为"禁止修改"的文件**
8. **不要引入任务书未提及的新依赖**
9. **不要删除任何现有文件**（除非任务书明确要求）
10. **不要执行任务书未列出的命令**
11. **接口契约文件（interface-contract.md）为只读**，不可修改
12. **命令执行超时 300 秒**，超时后记录到报告中

## 产出质量

13. 代码必须可编译、可运行
14. 不要生成占位符代码（`// TODO` 或 `throw new Error('not implemented')`）
15. 每个函数/方法都必须有实际实现
